B
Meta WhatsApp Business API — auto-reply to clients
developers.facebook.com
▼
💡 What this does
When a client WhatsApps KJ's number, Meta sends the message to your server, Claude replies, and the reply goes back through Meta to the client. You need 3 values: Access Token, Phone Number ID, and Verify Token.
⚠️ Requirement
You need a Facebook Business account and a phone number not already on WhatsApp (can be a new SIM or virtual number). Your existing personal WhatsApp number cannot be used.
1
Go to developers.facebook.com → My Apps → Create App
2
Select Business as app type → name it KJ Land Surveyors → Create
3
On your app dashboard → Add Product → find WhatsApp → click Set Up
4
Go to WhatsApp → API Setup. You'll see a test phone number already set up. Copy the Phone Number ID shown on screen.
5
Click Generate Access Token → copy it. Note: this temporary token expires in 24 hours. After deployment, generate a permanent token via System Users (see note below).
6
Make up any string for your Verify Token — e.g. kjlandsurveyors2026. You'll paste this same string in Meta's webhook config later.
🔑 Permanent Access Token (do this after deployment)
Meta Business Suite → Settings → System Users → Add → Admin role → Generate Token → select your app → grant whatsapp_business_messaging permission → copy the token. This never expires.
.env — paste your values
# Meta WhatsApp Business API WHATSAPP_TOKEN=EAAxxxxxxxxxxxxxxxxx WHATSAPP_PHONE_ID=1234567890123456 WHATSAPP_VERIFY_TOKEN=kjlandsurveyors2026 DIRECTOR_PHONE=254720397313
Facebook App created with WhatsApp product added
Phone Number ID copied
Access Token copied
Verify Token decided (e.g. kjlandsurveyors2026)

D
Google Sheets API — auto-sync operations data
console.cloud.google.com
▼
💡 What this does
Every new client, project, payment, and daily summary auto-writes a row to your Google Spreadsheet in real time. No manual data entry ever again.
1
Go to console.cloud.google.com → click Select a Project → New Project → name: KJ-LandSurveyors → Create
2
APIs & Services → Library → search Google Sheets API → click it → Enable
3
APIs & Services → Credentials → Create Credentials → Service Account → name: kj-sheets-agent → Role: Editor → Done
4
Click your new service account → Keys tab → Add Key → JSON → Download. Open the file and find client_email and private_key.
5
Go to sheets.google.com → create a new spreadsheet → name it KJ Land Surveyors Operations. Copy the long ID from the URL (the part between /d/ and /edit).
6
In the spreadsheet → Share → paste your client_email from the JSON → set role to Editor → Send. This lets your server write to it.
⚠️ Private Key Formatting
When pasting into .env, the private key must have \n replacing actual newlines, and must be wrapped in double quotes. The format is shown below.
.env — paste your values
# Google Sheets Service Account GOOGLE_SERVICE_ACCOUNT_EMAIL=kj-sheets-agent@kj-landsurveyors.iam.gserviceaccount.com GOOGLE_PRIVATE_KEY="-----BEGIN RSA PRIVATE KEY-----\nMIIEpAIBAAKCAQEA...\n-----END RSA PRIVATE KEY-----" GOOGLE_SPREADSHEET_ID=1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms

Deploy to Railway
Free tier · always-on · HTTPS included · takes ~55 minutes
⏱ ~55 min
💡 Why Railway?
Railway gives you a public HTTPS URL (required by Meta for webhooks), free tier with 500 hours/month, automatic deploys from GitHub, and built-in environment variable management. No server knowledge needed.
R
Create Railway Project & Deploy
railway.app
▼
1
Go to railway.app → Start a New Project → sign in with GitHub
2
Create a new GitHub repository called kj-ai-server (private is fine)
3
Upload your server files to the repo: server.js, generate_quote.py, package.json
4
On Railway → New Project → Deploy from GitHub repo → select kj-ai-server
5
Railway detects Node.js automatically and starts deploying. Wait ~2 minutes.
6
Go to Settings → Networking → Generate Domain. You'll get a URL like kj-ai-server-production.up.railway.app. Copy this — you need it for the next steps.
7
Add a Procfile to your repo with content: web: node server.js if Railway doesn't auto-detect your start command.
